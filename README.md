## What it does

Every day at 18:00 UTC, a GitHub Actions workflow scrapes news articles from [iDnes.cz](https://www.idnes.cz) (via RSS) and [Extra.cz](https://www.extra.cz) (pages 1–5 of `/tema/news/`, via HTML scraping). Articles accumulate in JSON files throughout the week. Every Monday, a second workflow sends the week's articles to the Groq API (Llama 3.3 70B), which produces a structured Czech-language summary. The static site is then rebuilt and published to GitHub Pages automatically.

No server. No database. No manual work.

## Pipeline

```
Every day 18:00 UTC — GitHub Actions
┌─────────────────────────────────────────────┐
│  iDnes RSS feed      →  XML parse           │
│  Extra.cz /tema/news →  cheerio (pages 1-5) │
│                                             │
│  Deduplicate by URL                         │
│  Append to data/articles-YYYY-Www.json      │
│  git commit & push                          │
└─────────────────────────────────────────────┘

Every Monday 09:00 UTC — GitHub Actions
┌─────────────────────────────────────────────┐
│  Load previous week's articles from JSON    │
│  Send to Groq API (Llama 3.3 70B)           │
│  Save HTML digest to data/digest-YYYY-Www   │
│  Rebuild static site → docs/               │
│  git commit & push → GitHub Pages live      │
└─────────────────────────────────────────────┘
```

## Tech stack

| Layer | Technology | Why |
|---|---|---|
| Scraping | `axios` + `fast-xml-parser` | RSS feed parsing for iDnes |
| Scraping | `axios` + `cheerio` + JSON-LD | HTML scraping + structured data for Extra.cz |
| Storage | JSON files + git | Zero infrastructure, full history, human-readable |
| AI | Groq API — Llama 3.3 70B | Free tier, high quality, fast inference |
| Site | Vanilla HTML/CSS | No build step, no dependencies, instant Pages deploy |
| CI/CD | GitHub Actions | Scheduled scraping, digest generation, auto-deploy |
| Tests | Node.js built-in `node:test` | No extra dependencies |

## Key engineering decisions

**Git as a database** — scraped articles are committed as JSON files. This gives full history, easy rollback, and zero infrastructure cost. GitHub Actions runners are ephemeral, so data must be persisted through git.

**Incremental deduplication** — each scrape run passes already-known URLs as a `Set` to the Extra.cz scraper. This skips HTTP detail requests for articles already stored, reducing both latency and external requests.

**JSON-LD for dates** — Extra.cz has no RSS feed. Article publish dates are extracted from `<script type="application/ld+json">` structured data blocks embedded in each article page, parsing both flat and `@graph`-format schemas.

**ISO 8601 week keys** — articles are bucketed by ISO week (`2026-W15`). The key algorithm anchors on Thursday, correctly handling year boundaries where e.g. Dec 29, 2025 belongs to week 2026-W01.

**Single entry point** — `src/index.js` handles all modes via CLI args (`--scrape`, `--digest`, `--build`), keeping GitHub Actions workflow files minimal and the codebase easy to reason about.

## Project structure

```
src/
  scrapers/
    idnes.js        RSS feed scraper
    extra.js        HTML scraper, pages 1-5 of /tema/news/, JSON-LD date extraction
  storage/
    store.js        File-based storage, ISO week key logic
  ai/
    summarize.js    Groq API call, prompt engineering
  index.js          Orchestrator, CLI dispatch, local cron

scripts/
  build-site.js     Static site generator (index + per-week pages)
  test-scrapers.js  Dev utility — print scraper output to console
  test-html.js      Dev utility — generate preview HTML from stored data

tests/
  unit/
    store.test.js          ISO week calculation, deduplication logic
  integration/
    idnes.test.js          Live iDnes scrape, shape validation
    extra.test.js          Live Extra.cz scrape, knownLinks skip logic

.github/workflows/
  scrape.yml        Daily scrape (cron 18:00 UTC)
  digest.yml        Weekly digest + site build (Monday 09:00 UTC)
  test.yml          Unit tests on every push to main

data/               Accumulated articles and digests (committed to git)
docs/               Generated static site served by GitHub Pages
```

## Running locally

```bash
git clone https://github.com/Hornok97/weekly-news-digest.git
cd weekly-news-digest
npm install
cp .env.example .env
# add your GROQ_API_KEY to .env
```

```bash
npm run scrape        # scrape articles for the current week
npm run digest        # generate AI digest for the previous week
npm run build         # rebuild the site from stored data
npm start             # run local cron scheduler (08:00 + Monday 09:00)
```

```bash
npm test              # unit tests — no network, ~150ms
npm run test:scrape   # integration tests — live network, ~45s
```

## GitHub Actions setup

1. Fork or clone the repo
2. Add secret `GROQ_API_KEY` in Settings → Secrets and variables → Actions
3. Enable GitHub Pages: Settings → Pages → Source: `main` branch, `/docs` folder
4. Workflows trigger automatically — or run manually via Actions → Run workflow
